# دليل إدارة المنتجات (Product Management Guide)

مرحباً بك في الدليل المرجعي الخاص بإدارة الكتالوج الرقمي لمنتجات `Smart Electricity`.
صُمم نظام المنتجات ليكون خفيفاً وسريعاً يعتمد بالكامل على ملفات `JSON`، لضمان تحميل فوري للمعلومات والأداء المتميز للعملاء.

---

## 1. الهيكلة العامة ومسارات العمل
تتوزع بيانات المنتجات في موقعك كالتالي:
- `data/products/` : المجلد الذي التخزن فيه ملفات الـ `JSON` الخاصة بمعلومات كل منتج على حدة.
- `assets/products/` : المجلد الذي توضع فيه صور المنتجات.
- `data/products_index.json` : القائمة المجمعة المرجعية، يستخدم السكربت لتوليدها وعرضها بسرعة فائقة.

---

## 2. كيفية إضافة منتج جديد (خطوة بخطوة)

لإضافة منتج جديد، لا تحتاج لأي خلفية برمجية معقدة. كل ما عليك فعله هو إتباع سلسلة من الخطوات البسيطة.

### الخطوة الأولى: إنشاء ملف المنتج
1. اذهب لمجلد `data/products/`
2. أنشئ ملفاً جديداً باسم يمثل المُعرف الفريد (يفضل أن يكون بالإنجليزية وبدون مسافات، استخدم الشَرطات التحتية أو العادية). مثال: `aqara-smart-switch.json`
3. قم بلصق القالب الأساسي التالي بداخله:

```json
{
  "id": "aqara-smart-switch",
  "brand": "Aqara",
  "technology": ["Zigbee", "WiFi"],
  "product_type": "Switch",
  "installation": "Wall mount",
  
  "title": {
    "en": "Aqara Smart Switch",
    "fr": "Interrupteur Intelligent Aqara",
    "ar": "مفتاح ذكي من أقارا"
  },
  
  "short_description": {
    "en": "Short text for catalog view...",
    "fr": "Texte court pour la vue catalogue...",
    "ar": "وصف قصير لواجهة الكتالوج..."
  },
  
  "description": {
    "en": "Long detailed text...",
    "fr": "Texte détaillé long...",
    "ar": "وصف شامل وتفصيلي للمنتج..."
  },
  
  "specs": {
    "en": {
      "Voltage": "220V",
      "Color": "White"
    },
    "fr": {
      "Tension": "220V",
      "Couleur": "Blanc"
    },
    "ar": {
      "الفولتية": "220V",
      "اللون": "أبيض"
    }
  },
  
  "images": [
    "/assets/products/aqara-smart-switch.jpg"
  ],
  
  "files": [
    {
      "name": "Installation Manual V1",
      "url": "https://example.com/manual.pdf"
    }
  ]
}
```

## 3. التصنيف والفلترة (Categorization)

لكي ينخرط المنتج بنجاح ضمن قائمة الفلاتر الجانبية في الصفحة الرئيسية للمنتجات، تأكد من استخدام المتغيرات التالية بعناية لكي تتطابق:

- **`brand`**: اسم الشركة المصنعة. (مثال: `EAE Technology`, `Fibaro`, `Aqara`). دقة الأحرف ستجعل الأجهزة المماثلة تندرج تحت فلتر واحد.
- **`product_type`**: صنف الجهاز وعائلته. (مثال: `Actuator`, `Sensor`, `Dimmer`, `Power Supply`).
- **`technology`**: مصفوفة (Array) للبروتوكولات المدعومة (مثال: `["KNX"]` أو `["Z-Wave", "Bluetooth"]`).
- **`installation`**: طريقة التثبيت الفيزيائية. (مثال: `DIN Rail mounting`, `Wall mount`, `Free standing`).

> [!IMPORTANT]
> **ترجمة الفلاتر الجديدة:** عند إدخال قيمة جديدة للتصنيفات (سواء في `product_type` أو `installation`) لم تكن مستخدمة من قبل، ستظهر بالإنجليزية في قائمة الفلاتر الجانبية حتى عند تغيير لغة الموقع. للحصول على ترجمة صحيحة، يجب إضافة مفتاح هذه القيمة في ملفات الترجمة: `ar.json` و `fr.json` (التي توجد في مجلد `assets/locales/`). 
> **قاعدة المفتاح:** تبدأ المفاتيح بـ `val_` مع استبدال أي مسافات بالشرطات السفلية (`_`). 
> مثال: لنوع `Switch`، أضف السطر `"val_Switch": "مفتاح ذكي"` في `ar.json`. ولنوع `Room Control Unit` أضف `"val_Room_Control_Unit": "وحدة تحكم الغرفة"`.

## 4. نظام الترجمة المدمجة (Localization)

يعتمد موقعك على بنية متعددة اللغات ليكون متاحاً لكافة زواره من مختلف الدول. لضمان ظهور المنتجات للجميع، تأكد من استيفاء حقول:  `"en"` و `"fr"` و `"ar"` داخل القوائم التالية:
- **`title`**: عنوان المنتج الذي سيظهر بخط بارز.
- **`short_description`**: الوصف القصير (المهم جداً). وهو النص الموجز الذي يظهر للمستخدم في "بطاقة المنتج" داخل الصفحة الرئيسية للمنتجات (الكتالوج) ويُعطي فكرة سريعة عن الجهاز.
- **`description`**: الوصف الطويل والشامل في صفحة العرض المخصصة للمنتج.
- **`specs`** (المواصفات التقنية - الجدول)

*نصيحة:* إذا لم يكن لديك ترجمة جاهزة، اترك أحدها بكلمة تعويضية أو أعد تضمين الإنجليزية فيها حتى لا تظهر بمقاييس فارغة.

## 5. إدارة المرفقات: تنزيل الملفات والصور

### الصور (`images`)
لإدراج صورة، احرص على:
1. تصغير حجم الصور قدر الإمكان باستخدام أبعاد جيدة وبأحجام لا تتعدى بضع مئات الكيلوبايتات لتسريع التنقل بالموقع.
2. وضع الصور في مسار `assets/products/`.
3. استدعاء الصورة في ملف `json` كالتالي:  
   `"images": ["/assets/products/your-image.png"]`

### التنزيلات والأدلة (`files`)
هذه الخانة مسؤولة عن إظهار الأيقونة الخاصة بالتنزيلات في صفحة المنتج. للالتزام بالمعايير الحالية:
- تأكد أن كل عنصر يمتلك اسم `"name"` يظهر للمستخدم، ورابط `"url"` يشير للملف نفسه.
- الأفضل أن تستخدم روابط تنتهي بـ `.pdf` لكتيبات التعليمات، أو `.knxprod` لملفات تعريف النظام الذكي. مثال:

```json
"files": [
  {
    "name": "Datasheet",
    "url": "https://www.eaetechnology.com/upload/dosyalar/Datasheet.pdf"
  },
  {
    "name": "ETS Product File",
    "url": "https://www.eaetechnology.com/upload/dosyalar/Device.knxprod"
  }
]
```

## 6. دمج ورفع المنتج (Build & Publish)

بمجرد الانتهاء من تجهيز ملف الـ `JSON` ومرفقاته، المنتج **لن يظهر في الصفحة** حتى يتم تحديث قاعدة البيانات الشاملة للموقع.
لا تقلق، الأمر آلي تماماً، لتوليد كل شيء قم بـ:

1. فتح نافذة الطرفية (Terminal) أو سطر الأوامر بداخل المجلد الخاص بموقعك `abdo7085.github.io`.
2. إدخال وتفعيل هذا الأمر المبرمج:
   ```bash
   python scripts/build_products.py
   ```
3. هذا الأمر الواحد يقوم بثلاث عمليات تلقائياً:
   - **بناء الفهرس** (`data/products_index.json`): يجمع كافة منتجاتك في ملف واحد خفيف لتصفح سريع.
   - **توليد صفحات SEO ثابتة**: ينشئ صفحة HTML مستقلة لكل منتج بكل لغة (في مجلدات `products/`, `fr/products/`, `ar/products/`). كل صفحة تحتوي على صورة المنتج ووصفه مباشرة في الكود، مما يجعل Google وWhatsApp وFacebook يقرؤون المعلومات فوراً.
   - **تحديث خريطة الموقع** (`sitemap.xml`): يضيف روابط المنتجات الجديدة تلقائياً ليجدها Google بسهولة.

> [!TIP]
> بعد تنفيذ الأمر أعلاه وظهور رسالة النجاح، اصبح منتجك جاهز! كل ما عليك هو رفعه (Push) إلى خوادم GitHub ليتفاعل معه عملائك حول العالم!

> [!IMPORTANT]
> **روابط المنتجات الجديدة:** أصبح لكل منتج رابط ثابت خاص به:
> - `https://smartelectricity.ma/products/<product-slug>.html` (الإنجليزية)
> - `https://smartelectricity.ma/fr/products/<product-slug>.html` (الفرنسية)
> - `https://smartelectricity.ma/ar/products/<product-slug>.html` (العربية)
> 
> هذه الروابط تُنشأ تلقائياً عند تشغيل أمر البناء. **لا تقم بتعديل الملفات في مجلدات `products/` يدوياً** — فهي تُعاد توليدها تلقائياً.

---

## 7. حذف منتج

عند حذف منتج، اتبع هذه الخطوات:
1. احذف ملف الـ JSON من `data/products/`.
2. احذف الصفحات الثابتة: `products/<slug>.html` و `fr/products/<slug>.html` و `ar/products/<slug>.html`.
3. شغّل أمر البناء: `python scripts/build_products.py` لتحديث الفهرس والـ sitemap.
4. اختيارياً: احذف صور المنتج من `assets/products/`.
