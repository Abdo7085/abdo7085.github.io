10 / Rules

What to keep, what to avoid.

These rules keep future implementation from drifting back into generic cool-gray UI or noisy store patterns.

Do

  • Use real assets: product photos, project media, and clean brand SVGs.
  • Route color and typography through `assets/brand.css` tokens.
  • Keep copper as the only dominant accent.
  • Test EN, FR, and AR layouts at desktop and mobile widths.
  • Use quote-request language instead of checkout or inventory promises.

Don't

  • Return to blue-gray neutrals or purple gradients.
  • Use full-pill buttons outside chips and badges.
  • Darken child buttons when hovering a whole card.
  • Hide brand/product/place identity below the first viewport.
  • Decorate with abstract blobs when real media can carry the page.
