## 1. هوية الموقع وتوجّهه وتركيزه

### من نحن
**S‑ELECTRICITY (Smart Electricity)** — شركة حلول المنزل الذكي والكهرباء في **تطوان، المغرب**. موقع ثابت (GitHub Pages) ثلاثي اللغة: **الإنجليزية / الفرنسية / العربية** (مع دعم RTL كامل).

### التركيز (Focus)
- **أتمتة المنزل الذكي** (KNX, Wi‑Fi, Zigbee, Matter, Thread).
- **المراقبة والأمن** (كاميرات، أقفال ذكية، أجراس فيديو، حسّاسات حركة).
- **الشبكات وتحسين الواي‑فاي** (Mesh).
- **التركيبات الكهربائية** وتصميم الإضاءة.
- **كتالوج منتجات** من ماركات موثوقة: Shelly · SONOFF · Reyee · EAE Technology · Dahua · Tuya · Zennio.

### الجمهور
ملّاك المنازل والمشاريع التجارية في المغرب — يبحثون عن خبير محلي موثوق يجمع بين الحرفة الكهربائية والتقنية الذكية.

### التموضع (Positioning)
> **"حرفة كهربائية أصيلة، بلمسة ذكية."**

الموقع ليس متجراً رخيصاً ولا واجهة تقنية باردة — بل **خبير دافئ وموثوق وراقٍ**. كل قرار بصري يخدم ثلاث كلمات: **دافئ · حِرَفي · موثوق**.

## 2. الاتجاه البصري المعتمد: دافئ / حِرَفي (Warm / Artisanal)

الإحساس المستهدف: نحاس وخشب داكن دافئ، ورق أبيض-رمادي هادئ («ستون» — منذ 2026-06-10، كان كريمياً قبلها)، نسيج خفيف، تفاصيل "مصنوعة باليد"، وتيبوغرافي بشخصية. نقيض الإحساس الـ"AI العام" (رمادي بارد، خطوط بلا هوية، تدرّجات بنفسجية، تخطيطات متوقّعة).

**مبادئ الاتجاه:**
1. **النحاس هو البطل** — لون أكسنت واحد حادّ، لا ألوان متعدّدة موزّعة بالتساوي.
2. **كل حياد بنبرة دافئة** — الأسطح الفاتحة رمادي «ستون» يميل قيد أنملة للدفء (R≥G≥B منذ 2026-06-10)، والنص إسبريسو؛ ممنوع الميل للأزرق البارد.
3. **نسيج لا لمعان** — حُبيبات (grain) خفيفة تعطي عمقاً ملموساً بدل أسطح مسطّحة.
4. **تخطيط بشخصية** — تركيب تحريري بأسلوب لوحة عيّنات (كل عنصر وحده مع وصف دقيق بجانبه)، تفاصيل editorial؛ تجنّب الأنماط المكرّرة والتداخل الذي يحجب العناصر.
5. **توازن EN/FR/AR** — أي تصميم يجب أن يصمد في العربية (RTL) كما في اللاتينية.

## 3. نظام الألوان (Color System)

### 3.1 النحاس — الهوية الأساسية (لا تتغيّر)
```css
--brand:       #b86c25;   /* النحاس الأساسي */
--brand-deep:  #9c5a1e;   /* hover / active أغمق */
--brand-pale:  #f5e7d8;   /* خلفيات tint كريمية-نحاسية */
--brand-glow:  rgba(184,108,37,0.45);  /* أكسنت الظلّ */
```

### 3.2 الطيف الداكن الدافئ (Warm‑dark — يبقى)
للأقسام/الهيرو الداكنة. **ليس** رمادياً بارداً.
```css
--warm-dark:       #1a140d;  /* الأساس */
--warm-dark-deep:  #0f0a05;  /* الأعمق — استخدمه كأطراف التدرّج مع grain */
--warm-dark-light: #3d2e1f;  /* الأفتح */
```

### 3.3 الحياد — عائلة «ستون» أبيض-رمادي بنبرة دافئة (محدَّث 2026-06-10)
**النص يبقى إسبريسو، والأسطح أبيض-رمادي «ستون» راقٍ** — بطلب المستخدم («اجعل الخلفية الرئيسية بالأبيض الرمادي… تناسق احترافي يليق بشركة عريقة»). رمادي أولاً، مع **همسة دفء فقط** (قنوات R≥G≥B) حتى يبقى النحاس والكروم الإسبريسو منتميَين للصورة نفسها. **أسماء التوكنات لم تتغيّر** (`--cream`/`--cream-deep` أسماء تاريخية) — كل الملفات تمرّ عبر `var()`، تغيّرت القيم فقط.
```css
--ink:        #2a1f16;  /* النص الأساسي — إسبريسو دافئ (بدل #1f2937) */
--ink-soft:   #6b5d4f;  /* نص ثانوي — تاوب دافئ (بدل #6b7280) */
--ink-faint:  #9b8b7a;  /* نص خافت */
--paper:      #fdfdfb;  /* سطح البطاقات — أبيض شبه نقي */
--cream:      #f1efec;  /* tints غائرة داخل البطاقات الورقية (خلفيات الصور/الخلايا) */
--cream-deep: #e5e3df;  /* كانفس الصفحة — رمادي فاتح «ستون» دافئ النبرة؛ أعمق بدرجة
                           واضحة من cream/paper حتى تبرز البطاقات والألواح ككائنات
                           (الدرس الأصلي: «البطاقة والخلفية لهما ألوان متشابهة جداً») */
--line:       rgba(42,31,22,0.12);  /* خطوط شعرية دافئة */
```

> **القاعدة:** ممنوع الرمادي **الأزرق/البارد**. حياد الأسطح رمادي «ستون» يميل قيد أنملة للدفء، وكل حياد يمرّ عبر هذه التوكنات.
> **تاريخ القيم:** قبل 2026-06-10 كانت العائلة كريمية/بيج (`#fffdf9 / #f4ede2 / #eae0cd`). حيثما وردت كلمة «كريمي» في بقية هذا الدليل عن أسطح فاتحة، فالمقصود توكنات `--paper`/`--cream` بقيمها الحالية (ستون).

## 4. التيبوغرافي (Typography)

| الدور | الخط | الاستخدام |
|------|------|-----------|
| **Display** | **Fraunces** (variable, optical sizing + wonk) | العناوين اللاتينية (EN/FR) — serif "قديم الطراز" دافئ بإحساس منحوت باليد |
| **Body + كل العربية** | **Readex Pro** | الجسم، الواجهات، **وكل عناوين العربية** |

```css
--font-display: 'Fraunces', Georgia, serif;        /* عناوين لاتينية */
--font-body:    'Readex Pro', system-ui, sans-serif; /* جسم + كل العربية */
--tracking-display: -0.022em;   /* tracking للعناوين الكبيرة */
--tracking-eyebrow:  0.22em;    /* للـ eyebrows الصغيرة uppercase */
```

**إعداد Fraunces للعناوين:** `font-optical-sizing: auto; font-variation-settings: "opsz" 144, "SOFT" 30, "WONK" 1;` — مع `em` بميل إيطالي ولون نحاسي للكلمات المميّزة داخل العنوان.

**قواعد حرجة:**
- **Fraunces لا يدعم العربية** → عناوين العربية تبقى **Readex Pro** (نظيف وناعم على Windows/Chrome). لا تستبدله بـ IBM Plex Arabic (مبكسل).
- **اختياري مستقبلاً:** خط عربي حِرَفي للعناوين العربية فقط (مثل Aref Ruqaa / Rakkas) إن أُريد شخصية للعربية أيضاً.
- يبقى السبب التاريخي لاختيار Readex Pro: عائلة واحدة تغطّي Latin + Arabic بتيبوغرافي موحّد.

## 5. المقاسات والحواف والظلال

### نصف القطر (Radius)
```css
--r-sm:   8px;
--r:      14px;
--r-btn:  16px;   /* الأزرار: شبه مقوّسة، لا pill */
--r-lg:   22px;   /* البطاقات والألواح */
--r-pill: 999px;  /* للوسوم/الشارات الصغيرة فقط — ليس للأزرار */
```

> **قاعدة الأزرار:** كل الأزرار **شبه مقوّسة** (`--r-btn` ≈ 16px للكبير، ~10px للصغير). **ممنوع** الـ pill الكامل للأزرار. الـ pill محجوز للوسوم (chips/tags) فقط.

### الظلال الدافئة (لا أسود بارد)
```css
--shadow-sm: 0 4px 14px -6px rgba(58,36,14,0.30);
--shadow:    0 16px 44px -18px rgba(58,36,14,0.45);
```

## 7. المكوّنات وأنماطها (Components & Patterns)

### 7.1 الأزرار (Buttons)
- **شبه مقوّسة** (`--r-btn`)، لا pill.
- **الزر الثانوي/المخطّط** (مثل *Add to quote*): افتراضياً **فاتح** (خلفية `--brand-pale` + نص `--brand-deep` + حدّ نحاسي).
- **يغمق (يمتلئ نحاسياً + نص أبيض) فقط عند التفاعل مع الزر نفسه** (`:hover` / `:active` / لمس) — **وليس** عند تمرير الماوس على البطاقة الحاوية. القاعدة: التأثير على العنصر الذي تلمسه، لا على حاويته.
- الزر الأساسي على الداكن: نحاس ممتلئ + ظلّ نحاسي خفيف.

### 7.2 بطاقة المنتج (Product Card) — النمط المعتمد
بُنيت بالكامل من مكوّنات الموقع الحقيقية:
- **الصورة** على خلفية كريمية مع توهّج نحاسي خفيف من الأعلى.
- **لا eyebrow لاسم الماركة فوق العنوان (2026-06-06):** العلامة تُقرأ من الشعار في الذيل فقط؛ أُزيل سطر اسم الماركة (`.prod-card-brand`) الذي كان فوق العنوان من المُصيّرات الثلاثة (`products.js` / `product-detail.js` / `brand-detail.js`).
- **شعار الماركة + زر "+ Add to quote" = صفّ واحد يتقلّص معاً (2026-06-06):** يجلسان **أسفل البطاقة** في طرفين متقابلين، **بلا التفاف ولا تكديس عمودي**. يصغُران معاً مع عرض البطاقة عبر وحدات `cqi` (الشعار `clamp(38px,24cqi,72px)`؛ الزر font/padding بـ`cqi` ونصّه `nowrap`) — يتطلّب `container-type: inline-size` على `.prod-card`. الشعار **بصندوق موحّد** (`object-fit:contain`) بحدّ أقصى ~72×22px على البطاقات العريضة فتُقرأ كل الشعارات بحجم واحد. `logo: null` → يُعرض اسم الماركة نصّاً.
- **شريط البروتوكولات:** بين الصورة والعنوان/الوصف — أيقونة + اسم لكل بروتوكول، تُقرأ من `product.technology[]` وتعيد استخدام **قاموس الأيقونات الرسمي** في [`product-detail.js`](../assets/product-detail.js) (~L319): Wi‑Fi · Bluetooth · **KNX (نصّ عريض)** · **Zigbee** (شعار رسمي) · **Matter** (شعار رسمي) · **Thread** · Ethernet · Z‑Wave. الشريط يختفي إن لم توجد بروتوكولات.
- **تأثير الـ hover:** **أقواس نحاسية شبه مقوّسة في زوايا الصورة** (تؤطّر منطقة صورة المنتج فقط — **لا البطاقة كاملة**) تنطبق في بؤرة التركيز — **بديل مميّز** عن "الخط العلوي" الشائع (الذي رُفض لكونه عاماً). تُستخدم نفس الأقواس على بطاقة الماركة لكن حول منطقة الشعار فقط.
- **مستقيمة** (بدون ميلان/دوران)، وتقف وحدها في "لوحة العيّنات" مع وصف دقيق بجانبها — بلا تداخل (انظر §7.3).
- **معلومات قابلة للصيانة فقط:** ممنوع عبارات مثل "In stock" (لا تتبّع مخزون) — استخدم سمات ثابتة (ماركة، بروتوكول).

### 7.3 التخطيط المميّز — "لوحة العيّنات" (Specimen Sheet)
لتفادي التخطيطات المتوقّعة، الأقسام البطلة تتبع تركيباً تحريرياً بأسلوب "ورقة عيّنات":
- **كل عيّنة وحدها + وصف دقيق بجانبها:** كل مكوّن (لوحة مميّزة، بطاقة منتج) يقف في صفّه الخاص، وإلى جانبه شرح تحريري دقيق (رقم الشكل، eyebrow، عنوان، فقرة قصيرة، وقائمة سمات بنقاط نحاسية). **بلا تداخل** — حتى لا يغطّي عنصرٌ جزءاً من آخر. على الموبايل تتراصّ العيّنة فوق وصفها.
- **رقم عملاق مفرّغ** بحدّ نحاسي ينزف خلف عنوان القسم.
- **لصيقة عمودية** على الحافة + ترقيم الأشكال `Fig. 01 / Fig. 02` (شكل ٠١ / ٠٢) داخل الوصف بأسلوب كتالوج.
- **عمود نحاسي** على حافة الألواح الداكنة + علامات `Plate Nº` (عيّنة رقم).
- **الـ CTA = تذكرة مختومة:** خط تثقيب متقطّع + ثقوب + ختم دائري `S·E` بدل بانر مسطّح.

> **ملاحظة (تحديث 2026-05-31):** أُلغي نمط **التداخل** (البطاقة تتسلّق فوق اللوحة) لأنه كان يغطّي جزءاً من اللوحة المميّزة؛ استُبدل بفصل كل عيّنة في صفّها مع وصف دقيق بجانبها. المعاينة تجسّد هذا.

### 7.4 الحركة (Motion)
- **دخول مُنسّق (staggered reveal):** ظهور تدريجي خفيف للعناصر عند التحميل (`animation-delay`).
- **احترام `prefers-reduced-motion`** على مستوى الموقع — يُلغى كل تحريك للمستخدمين الذين يطلبون ذلك.

### 7.5 حقل البحث في الكتالوج (Catalog Search Field)
يجلس على هيرو الكتالوج الداكن الدافئ، ومُصمَّم كـ**"حوض غائر دافئ"** لا حبّة (pill) باردة:
- **الشكل:** شبه مقوّس (`14px` — يوافق خطوة `--r` في §5) بـ**ارتفاع أدنى `50px`** (حوض ممتلئ مطابق للمعاينة). **لا** pill كامل، التزاماً بقاعدة الأزرار/الحواف.
- **السطح:** مسحة دافئة شفّافة (`rgba(245,231,216,0.06)`) + حدّ شعري دافئ (`…,0.22`) + `backdrop-filter: blur`. ممنوع الأبيض البارد.
- **العمق (tactile):** ظلّ داخلي غائر + لمعة حافة علوية (`inset`) — يُحسّ كحقلٍ يُكتب فيه، لا سطحٍ مسطّح.
- **العلامة النحاسية:** أيقونة البحث **نحاسية ناعمة في الراحة** (`rgba(184,108,37,0.85)`) ليُقرأ الأفوردانس فوراً، وتتوهّج إلى `--brand` الكامل عند التركيز.
- **الحالات:** *hover* يُدفئ الحدّ والخلفية؛ *focus* = حدّ نحاسي + حلقة (`--ring-soft`) + توهّج نحاسي رافع خفيف (`--brand-glow`)، مع هالة نحاسية خلف الأيقونة وزر مسح ✕ نحاسي.
- **placeholder:** دافئ ومقروء (`rgba(245,231,216,0.5)`)، لا رمادي خافت.
- **RTL:** خصائص منطقية بالكامل (`padding-inline-start` / `inset-inline-start`) — تنعكس تلقائياً بلا تجاوزات `[dir="rtl"]`.

> **المصدر الحيّ:** `.prod-search-input` في [`assets/products.css`](../assets/products.css) (مُطبَّق فعلاً — أول مكوّن حيّ بالاتجاه الدافئ). **المعاينة:** قسم «٠٥ / Catalog» في [`design-system-preview.html`](design-system-preview.html).

### 7.6 صفحة معلومات المنتج — "اللوح الورقي" (Product Detail Plate) — 2026-06-10
مطابقة لقسم «05 / Product detail» في المعاينة. **المصدر الحيّ:** كلاسات `pd-*` في `products.css` (+ صفّ الكمية في `cart.css`)، والماركأب في `product-detail.js`:
- **الكانفس كريمي** (يأتي من قاعدة `body:not(.home-page)` الموحّدة في `projects.css` — §10 درس 11)، وفوقه **لوح ورقي واحد** (`.pd-layout`: paper + حدّ شعري + `--r-xl` 30px + ظلّ دافئ) يضمّ المعرض والعمود المعلوماتي معاً.
- **المعرض:** thumbs مربّعة 72px (كريمي، نشطة بحدّ نحاسي + حلقة `--ring-soft`) + مسرح صورة بالخلفية الغائرة الموحّدة (توهّج نحاسي radial + `--cream` — نفس خلفية صور بطاقات الكتالوج) وزاوية `--r-lg`. سلوك التبديل والـ sticky كما هو.
- **العمود المعلوماتي** (grid بإيقاع gap موحّد): eyebrow «**ماركة / نوع**» بشرطة تحريرية (الماركة تبقى رابطاً لصفحتها) ← عنوان **Fraunces** (العربية Readex Pro) ← الوصف ← **خلايا سمات** (`.pd-feature` بخلفية `--cream`): «البروتوكولات» بأيقونات القاموس الموحّد + «التركيب» ← صفّ الكمية + زر الطلب (نحاسي بـ `--r-btn` وتوهّج) + تلميح واتساب ← **الملفات كأزرار هادئة** (btn-quiet). عنوانا «Specifications/Downloads» لصيقتان صغيرتان (`.pd-block-label`).
- **جدول المواصفات = لوح ورقي مستقل بعرض كامل تحت اللوح الرئيسي** (`.pd-specs-section` بنفس لغة اللوح: paper + `--r-xl` + ظلّ؛ صفوف شعرية بلا zebra). السبب: جداولنا الحقيقية 12–18 صفاً — وضعها داخل العمود (كما في المعاينة بثلاثة صفوف فقط) كان يترك فراغاً ورقياً ضخماً بجانبها.
- **قاموس أيقونات البروتوكولات وُحِّد:** `relProtoMarkup()` في `product-detail.js` يخدم خلايا السمات وبطاقات Related معاً (أُزيل قاموس الدوائر 38px القديم).

## 9. خريطة التطبيق — ما تمّ تطبيقه حيّاً (2026-06-01)

تمّ تنفيذ الاتجاه الدافئ على **٤ أنواع صفحات**: الكتالوج، معلومات المنتج، الماركات، معلومات الماركة. التفاصيل الدقيقة (مكان كل تغيير) لإعادة الاستخدام أو التوسعة:

1. **التوكنات الدافئة** → `assets/brand.css` (المصدر الواحد): `--ink #2a1f16`, `--ink-soft #6b5d4f`, `--ink-faint`, `--paper #fffdf9`, `--cream #f4ede2`, `--line`، + `--font-display` (Fraunces) + سلّم نصف القطر + ظلال دافئة. `--ink-muted` بقي **alias** لـ`--ink-soft` حتى لا تنكسر مراجع قديمة.

3. **التوكنات المحلية الباردة** → في `products.css` و`brands.css` كانت `--prod-border:#e5e7eb` و`--brand-text-*` رماديات باردة **بلا `var()`**؛ أُعيد ربطها بتوكنات `brand.css`. (راجع §10 — أكبر مصدر "رمادي متسرّب".)
4. **شعارات الماركات** → حقل `logo` في `data/brands/*.json` → `/assets/brands/<slug>.svg` ثم `python scripts/build_brands.py`. الحجم موحّد عبر صندوق `contain` ثابت (`.prod-card-logo`).
5. **بطاقة المنتج** (`products.js` `renderCard` + `brand-detail.js` + `product-detail.js` related): شريط بروتوكولات (`protoMarkup`) + عنوان Fraunces + أسفل: شعار + زر "+ Add to quote" فاتح. الستايل في `products.css` (`.prod-card-*`)، مشترك بين الثلاثة.
6. **أقواس الزوايا النحاسية** → على **حاوية الصورة فقط** (`.prod-card-img-wrapper::before` / `.brand-card-logo-area::before`)، لا البطاقة كاملة.
7. **`python scripts/build_all.py`** لنشر القوالب على `fr/`, `ar/`, `products/`, `projects/` (CSS/JS/locales تُخدَم مباشرة بلا بناء).
8. **مراجعة بصرية** يدوية عبر EN/FR/AR + موبايل/ديسكتوب (المستخدم يتحقّق — لا اختبار آلي).
9. **صفحة معلومات المنتج — لوح المعاينة (2026-06-10)** → راجع §7.6. الملفات: `product-detail.js` (ماركأب + `body.pd-page`) و`products.css` (كلاسات `pd-*`) و`cart.css` (صفّ الكمية الدافئ) + مفتاحا locale جديدان `pd_protocols`/`pd_installation` + توكنان جديدان في `brand.css`: `--line-strong` و`--r-xl`.

## 10. دروس مستفادة من جولة التطبيق (اقرأها قبل أي توسعة — توفّر وقتاً)

1. **«المعاينة أولاً» قاعدة ملزمة.** أي مكوّن جديد يُبنى في `design-system-preview.html` (EN+AR) ويُعتمد بصرياً **قبل** لمس الملفّات الحيّة. (تفصيل في ذاكرة المشروع: feedback_preview_first.)
2. **مسار شعارات الماركات هو `/assets/brands/<slug>.svg` — لا `/assets/brands/SVG/`.** المجلّد الفرعي `SVG/` **غير موجود**؛ مسارٌ خاطئ كلّف جولة كاملة (شعارات مكسورة 404 في المعاينة). تأكّد من المسار الفعلي قبل الإحالة.
3. **توحيد حجم الشعارات لا يكون بـ`max-width/height`** — ملفّات SVG بنسب مختلفة جداً (Shelly 3.6:1، Reyee كان مربّعاً 1:1، Zennio 4.6:1). الحلّ: **صندوق ثابت** (`width/height` + `object-fit:contain`). وإن كان شعار يبدو أصغر بكثير، **السبب غالباً فراغ داخلي في الـ`viewBox`** → قُصّ الـ`viewBox` لحدود الحبر (فعلناه لـ Reyee و EAE). لا تعالجه بـ CSS لكل ماركة على حدة.
4. **الرماديات الباردة تتسرّب من توكنات محلية بلا `var()`.** عند تدفئة الموقع، فتّش كل ملف عن `#1f2937 #6b7280 #374151 #475569 #cbd5e1 #94a3b8 #e2e8f0 #f1f5f9 #f8fafc #d1d5db rgba(15,23,42…)` — لا تكتفِ بتعديل `brand.css`.
5. **زر "Add to quote" داخل بطاقة-رابط `<a>`:** أضف `onclick="event.preventDefault();event.stopPropagation();if(window.Cart)window.Cart.add(id,1)"` — معالِج `data-trigger="cart-add"` العام موجود لكنه **في الـSPA bundle المُصغّر** (لا في `cart.js`)، فلا تعتمد عليه؛ الـAPI العامة `window.Cart.add(id, qty)` مضمونة.
6. **انتبه لتعارض `cart.css`:** كان فيه `.prod-card-add` قديم (زر أيقونة عائم مطلق فوق الصورة) يُحمَّل **بعد** `products.css` فيتغلّب عليه. عند إعادة تصميم زر، احذف القاعدة القديمة من `cart.css` لا فقط من `products.css`.
7. **كتل `@media` قد تُنسّق كلاسات محذوفة.** بعد إعادة بناء بطاقة، نظّف breakpoint الموبايل من قواعد ميتة (`.prod-card-tech`, `.prod-card-arrow`) وأضف قواعد العناصر الجديدة (أسفل البطاقة يتراصّ عمودياً + زر بعرض كامل تحت 640px).
8. **`build_all.py` يُعيد توليد `fr/ ar/ products/ projects/`** من القوالب الجذرية — لا تعدّلها يدوياً. لكن `assets/*.css|js` و`locales/*.json` تُخدَم مباشرة (تظهر فوراً، لا بناء).
9. **عند الدفع:** استبعد `docs - Copy/` (نسخة احتياطية). `docs/` نفسه متتبَّع (فيه المعاينة + الدليلان).
10. **تحقّق بعد كل تعديل CSS:** توازن الأقواس `{ }` (سكربت بايثون سريع)، وأن المعاينة والحيّ بقيا متطابقَين.
11. **توحيد الانسجام اللوني (2026-06-10) — أين يسكن كل لون «هيكلي»:** بعد شكوى «تناسق الألوان سيئ» تبيّن أن الموقع كان يخلط ٣ عائلات حياد: كروم أسود صرف، كانفس أبيض في صفحات وكريمي في أخرى، وحساء بيجات متقاربة غير متطابقة (`#f8f1e7` `#faf5ee` `#fbf5ee` `#eee0d0` `#efe6d9`…). القواعد الآن:
    - **الكروم الداكن** (نافبار/فوتر/قائمة اللغة/أقسام داكنة): عائلة الإسبريسو فقط. في Tailwind المُجمّع `index-CJ3jXuVd.css`: `.bg-secondary`/`.via-secondary`/`.to-secondary` = `#0f0a05`، `.from/to-gray-900` = `#1a140d`، `.text-secondary` = `--ink`، `.bg-gray-50/100` دُفّئتا، وحتى `border-color` الافتراضي في الـ preflight صار دافئاً. قاعدتا فرض النافبار في `products.css`/`projects.css` = `rgba(15,10,5,0.96)`.
    - **الكانفس**: قاعدة واحدة `body:not(.home-page) { background: var(--cream-deep) }` تسكن في **`projects.css`** (لأنه محمَّل في كل القوالب الجذرية بما فيها الرئيسية). حاويات الصفحات (`#custom-*-wrapper/container`) كلها `transparent` — لا توجد خلفية بيضاء على مستوى الحاوية إطلاقاً (يمنع «عمود أبيض» على الشاشات > 1280px). **سلّم الحياد الفاتح ثلاثي:** كانفس `--cream-deep` (الأعمق) ← tints داخلية `--cream` ← أسطح `--paper` (الأفتح) — الكانفس يجب أن يكون أعمق درجة من كل ما فوقه وإلا ذابت البطاقات فيه.
    - **الخلفية الغائرة الموحّدة** لمناطق الصور/الشعارات (بطاقة المنتج، معرض صفحة المنتج، منطقة شعار الماركة): `radial-gradient(120% 120% at 50% 0%, rgba(184,108,37,0.08), transparent 60%), var(--cream)` — درجة واحدة، لا بيجات مخصّصة لكل مكوّن.
    - **التوكنات المحلية** `--cart-*`/`--fs-*`/`--proj-*` كلها تمرّ عبر توكنات `brand.css` الآن (كانت `#e5e7eb`/`#1a1a1a`/`#fafafa` باردة بلا `var()`).
